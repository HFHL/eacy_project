# 升级流程

## 适用场景

线上 EACY 升级到新版本（拉取新代码、执行 migration、重启进程）。首次部署见 [[首次部署手册]]。

## 前置条件

- [ ] 已完成 [[备份与恢复]] 描述的 DB 备份（**强烈建议**）
- [ ] 已 review 本次升级的 commit 范围，确认是否包含 Alembic migration（看 `backend/migrations/versions/` 新增文件）
- [ ] 已确认是否包含 env 新增字段（参考 [[环境变量清单]]）
- [ ] 已通知相关方升级窗口

## 执行步骤

### 1. 拉取代码

```bash
cd /opt/eacy_project
git fetch --all
git status              # 确认无未提交修改
git pull --rebase
```

### 2. 更新依赖

只有 `backend/pyproject.toml` / `frontend_new/package.json` 变化时执行。

```bash
# 后端
cd /opt/eacy_project/backend
poetry install

# 前端
cd /opt/eacy_project
npm install
```

### 3. 更新 `.env`（如有新增字段）

参考新版本的 `DEPLOYMENT_RUNBOOK.md` 和 [[环境变量清单]] 增补。**不要直接覆盖** `.env`，逐项 diff。

### 4. 滚动升级顺序

> [!warning] 顺序：先停 worker → 跑 migration → 重启 API → 起 worker → 切前端
> 反顺序会出现"新 worker 用旧表结构、旧 API 用新表"的窗口期。

#### 4.1 停止 Celery worker（避免长任务读到迁移中途的 schema）

```bash
sudo systemctl stop eacy-celery-ocr eacy-celery-metadata eacy-celery-extraction
```

观察队列在 Redis 是否仍有未消费任务：

```bash
redis-cli -n 1 llen ocr metadata extraction
```

若有大量待处理任务，等其在停止前自然消费完，或可接受短暂积压。

#### 4.2 执行数据库迁移（migration 执行点）

```bash
cd /opt/eacy_project/backend
poetry run alembic current        # 记录当前版本
poetry run alembic upgrade head
```

> [!warning] migration 失败时
> 见 [[常见故障排查#migration失败回滚]]。**不要**在生产盲跑 `alembic downgrade`，多数 EACY migration 未实现安全 downgrade。

#### 4.3 重启后端 API

```bash
sudo systemctl restart eacy-backend
sudo systemctl status eacy-backend
journalctl -u eacy-backend -f --since "1 min ago"
```

确认 API 启动成功后再进入下一步。

#### 4.4 重启 Celery worker

```bash
sudo systemctl start eacy-celery-ocr eacy-celery-metadata eacy-celery-extraction
sudo systemctl status eacy-celery-ocr eacy-celery-metadata eacy-celery-extraction
```

Worker 重启策略说明：

- 三个 worker 是**独立 systemd unit**，可单独重启而不影响其他队列
- 仅 LLM 配置变更时，可只重启 `eacy-celery-metadata` 与 `eacy-celery-extraction`
- 仅 TextIn 配置变更时，可只重启 `eacy-celery-ocr`
- Celery 默认会等当前任务结束后退出，强杀会导致任务回到 broker 重新派发

#### 4.5 前端构建与发布

```bash
cd /opt/eacy_project
npm run build -w xidong-crf-prototype-new
sudo nginx -t && sudo systemctl reload nginx
```

新 `dist` 写入后 Nginx 立即生效，无需停服。

## 验证方式

- [ ] `systemctl status eacy-backend eacy-celery-*` 全部 active
- [ ] `curl -I http://127.0.0.1:8000/docs` 返回 200
- [ ] `alembic current` 输出与 `alembic heads` 一致
- [ ] 上传 / 抽取一次小样本，全流程 OK
- [ ] 前端登录后 hard refresh，无 chunk hash 加载失败

## 回滚方式

### 代码回滚

```bash
cd /opt/eacy_project
git log --oneline -5
git reset --hard <上一个稳定 commit>
# 重复升级步骤 2~5
```

### Migration 回滚

> [!warning] 仅在确认迁移脚本实现了 downgrade 时执行
> 多数 EACY migration 是 forward-only。若没有 downgrade，需要从 [[备份与恢复]] 的快照恢复。

```bash
cd /opt/eacy_project/backend
poetry run alembic downgrade <previous_revision>
```

### Worker / API 回滚

`git reset` 之后重新 `systemctl restart`。
