# 环境变量清单

> 全量 env 配置项来自 [`backend/core/config.py`](../../backend/core/config.py) 的 `Config` 类。`.env` 文件支持两个位置：项目根 `.env` 或 `backend/.env`（前者优先）。`ENV=local|test|prod` 决定使用的 `Config` 子类，仅 `ProductionConfig` 默认 `DEBUG=false`。

> [!warning] 变更 env 后必须重启相关进程
> JWT / DB / Redis / Celery broker / OSS / TextIn / LLM 配置都被进程在启动时读取并缓存。改 env 不会热生效，必须重启 [[首次部署手册|API]] 与 [[Celery任务运维|对应 Celery worker]]。

---

## 一、基础

| 变量名 | 必填 | 默认 | 说明 | 涉及模块 |
|---|---|---|---|---|
| `ENV` | 否 | `development`（`get_config` 默认 `local`） | 运行环境标识，影响选用的 `Config` 子类（`local` / `test` / `prod`）。 | 全局 |
| `DEBUG` | 否 | `True`（`ProductionConfig` 改为 `False`） | 调试开关。生产必须为 `false`。 | API |
| `APP_HOST` | 否 | `0.0.0.0` | `main.py` Uvicorn 监听地址。 | API |
| `APP_PORT` | 否 | `8000` | API 监听端口；`start-all.ps1` 读取此值清端口。 | API |
| `SENTRY_SDN` | 否 | `""` | Sentry DSN 占位。当前未在仓库内启用，保留接入位 (TBD)。 | API |

## 二、数据库

| 变量名 | 必填 | 默认 | 说明 | 涉及模块 |
|---|---|---|---|---|
| `DATABASE_URL` | 推荐 | `None` | 统一数据库 URL；若设置，会覆盖默认的 `WRITER_DB_URL` / `READER_DB_URL`。生产使用 `postgresql+asyncpg://...`。 | API / Worker / Alembic |
| `WRITER_DB_URL` | 否 | `mysql+aiomysql://fastapi:fastapi@localhost:3306/fastapi` | 写库 URL。仅在显式区分读写库时设置。 | API / Worker |
| `READER_DB_URL` | 否 | 同上 | 读库 URL。 | API / Worker |
| `DB_POOL_SIZE` | 否 | `1` | SQLAlchemy 连接池大小。 | API / Worker |
| `DB_MAX_OVERFLOW` | 否 | `1` | 连接池溢出连接数。 | API / Worker |
| `DB_POOL_TIMEOUT` | 否 | `5` | 等待连接超时（秒）。 | API / Worker |
| `DB_POOL_RECYCLE` | 否 | `600` | 连接回收时间（秒）。 | API / Worker |

> [!info] 关于连接池
> 默认值偏小，生产应按"`(API worker 数 + Celery worker 并发数)` × `pool_size`"评估总连接数与 DB `max_connections` 关系。见 [[常见故障排查#数据库连接耗尽]]。

## 三、Redis

| 变量名 | 必填 | 默认 | 说明 | 涉及模块 |
|---|---|---|---|---|
| `REDIS_HOST` | 否 | `localhost` | 业务缓存 Redis 主机。 | API / Worker |
| `REDIS_PORT` | 否 | `6379` | 业务缓存 Redis 端口；`scripts/daemon-start.sh` 启动前会探测此端口。 | API / Worker |

## 四、Celery

| 变量名 | 必填 | 默认 | 说明 | 涉及模块 |
|---|---|---|---|---|
| `CELERY_BROKER_URL` | 推荐 | `redis://localhost:6379/1` | Celery broker。生产 `deploy/production` 建议改用 DB 11 隔离。 | API / Worker |
| `CELERY_BACKEND_URL` | 推荐 | `redis://localhost:6379/2` | Celery result backend；亦接受别名 `CELERY_RESULT_BACKEND`。生产 DB 12。 | API / Worker |
| `CELERY_TASK_ALWAYS_EAGER` | 否 | `false` | 调试用同步执行。生产严禁置 `true`。 | API / Worker |

队列与任务名定义在 [`celery_app.py`](../../backend/app/workers/celery_app.py)：

| 队列 | 任务名 |
|---|---|
| `ocr` | `eacy.ocr.process_document_ocr` |
| `metadata` | `eacy.metadata.extract_document_metadata` |
| `extraction` | `eacy.extraction.process_extraction_job` |

## 五、对象存储（OSS）

| 变量名 | 必填 | 默认 | 说明 | 涉及模块 |
|---|---|---|---|---|
| `DOCUMENT_STORAGE_PROVIDER` | 否 | `oss` | 当前仅支持 `oss`。 | API / OCR Worker |
| `OSS_ACCESS_KEY_ID` | 是（生产） | `None` | 阿里云 AccessKey ID。 | API / OCR Worker |
| `OSS_ACCESS_KEY_SECRET` | 是（生产） | `None` | 阿里云 AccessKey Secret。 | API / OCR Worker |
| `OSS_BUCKET_NAME` | 是（生产） | `None` | OSS Bucket 名。 | API / OCR Worker |
| `OSS_ENDPOINT` | 是（生产） | `None` | OSS 接入域名，如 `oss-cn-shanghai.aliyuncs.com`。 | API / OCR Worker |
| `OSS_REGION` | 是（生产） | `None` | OSS Region，如 `cn-shanghai`。 | API / OCR Worker |
| `OSS_BASE_PREFIX` | 否 | `documents` | 对象路径前缀。 | API |
| `OSS_PUBLIC_BASE_URL` | 否 | `None` | 自定义公共域名；为空时使用 bucket endpoint 拼接。 | API |

## 六、TextIn OCR

| 变量名 | 必填 | 默认 | 说明 | 涉及模块 |
|---|---|---|---|---|
| `TEXTIN_APP_ID` | 是（启用 OCR） | `None` | TextIn 应用 ID。 | OCR Worker |
| `TEXTIN_SECRET_CODE` | 是（启用 OCR） | `None` | TextIn 应用 Secret。 | OCR Worker |
| `TEXTIN_API_URL` | 是（启用 OCR） | `None` | TextIn 接口地址，例如 `https://api.textin.com/ai/service/v1/pdf_to_markdown`。 | OCR Worker |
| `TEXTIN_TIMEOUT_SECONDS` | 否 | `120.0` | OCR 单次请求超时（秒）。 | OCR Worker |

## 七、LLM（OpenAI 兼容）

| 变量名 | 必填 | 默认 | 说明 | 涉及模块 |
|---|---|---|---|---|
| `OPENAI_API_KEY` | 是（启用 LLM） | `None` | LLM 鉴权 key。 | Metadata / Extraction Worker / API |
| `OPENAI_API_BASE_URL` | 否 | `https://api.openai.com/v1` | OpenAI 兼容接口 base URL。 | 同上 |
| `OPENAI_MODEL` | 否 | `gpt-4o-mini` | 默认模型名。 | 同上 |
| `METADATA_LLM_TIMEOUT_SECONDS` | 否 | `120.0` | 元数据 LLM 超时。 | Metadata Worker |
| `METADATA_LLM_TEMPERATURE` | 否 | `0.0` | 元数据 LLM 温度。 | Metadata Worker |
| `METADATA_LLM_ENABLE_RULE_FALLBACK` | 否 | `true` | LLM 失败时是否回落到规则提取。 | Metadata Worker |
| `EXTRACTION_LLM_TIMEOUT_SECONDS` | 否 | `180.0` | 字段抽取 LLM 超时。 | Extraction Worker |
| `EXTRACTION_LLM_TEMPERATURE` | 否 | `0.0` | 字段抽取 LLM 温度。 | Extraction Worker |

## 八、鉴权

| 变量名 | 必填 | 默认 | 说明 | 涉及模块 |
|---|---|---|---|---|
| `ENABLE_AUTH` | 是（生产） | `false` | 是否启用 JWT 鉴权。生产必须 `true`。 | API |
| `JWT_SECRET_KEY` | 是（生产） | `fastapi` | JWT 签名 key。生产必须替换为高熵随机字符串。 | API |
| `JWT_ALGORITHM` | 否 | `HS256` | JWT 签名算法。 | API |

> [!warning] 默认 `JWT_SECRET_KEY=fastapi` 是不安全占位
> 上线前必须替换；变更后所有已签发 token 失效，用户需要重新登录。

## 九、业务开关

| 变量名 | 必填 | 默认 | 说明 | 涉及模块 |
|---|---|---|---|---|
| `DOCUMENT_OCR_AUTO_ENQUEUE` | 否 | `true` | 上传文档后是否自动入 OCR 队列。关闭后需要通过 API 手动触发。 | API |
| `EACY_EXTRACTION_STRATEGY` | 否 | `simple` | 抽取策略；可选 `simple` / `llm` / `agent`（具体值以 [`extraction_service.py`](../../backend/app/services/extraction_service.py) 为准）。 | Extraction Worker |

## 修改 env 后需要重启的进程

| 变更 | 必须重启 |
|---|---|
| `JWT_*` / `ENABLE_AUTH` | API |
| `DATABASE_URL` / `DB_*` | API + 所有 Celery worker |
| `CELERY_*` / `REDIS_*` | API + 所有 Celery worker |
| `OSS_*` | API + OCR worker |
| `TEXTIN_*` | OCR worker |
| `OPENAI_*` / `METADATA_LLM_*` / `EXTRACTION_LLM_*` | metadata / extraction worker（也建议重启 API） |
| `DOCUMENT_OCR_AUTO_ENQUEUE` | API |
| `EACY_EXTRACTION_STRATEGY` | extraction worker |

详细操作见 [[首次部署手册]] 与 [[升级流程]]。
