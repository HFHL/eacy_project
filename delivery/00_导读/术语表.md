# 术语表

> 项目内反复出现的概念，统一定义。**遇到歧义先来这里**。后续业务域文档发现新术语应回填到这里。

## 业务术语

| 术语 | 英文 / 代号 | 含义 |
|---|---|---|
| 病例 | Patient | 一位被纳入平台的患者实体，承载其所有文档与 EHR 数据。**与"科研项目"是多对多关系**。 |
| 病例池 | PatientPool | 平台中**全量病例**的总集合，未必都纳入了具体科研项目。 |
| 文档 | Document | 上传到平台的原始医疗文件（PDF、扫描件等），可关联到病例。 |
| OCR | OCR | 光学字符识别。EACY 通过 TextIn OCR 把扫描件转为带坐标的结构化文本。 |
| EHR | Electronic Health Record | 电子病历，本项目中指**结构化后的患者临床字段集合**。 |
| CRF | Case Report Form | 病例报告表，科研中预先定义的字段表单。EACY 中由 Schema 模板表达。 |
| Schema 模板 | SchemaTemplate | 一组字段的可视化定义，支持嵌套分组、枚举、类型约束。模板有版本（SchemaTemplateVersion）。 |
| Schema 字段规划 | Schema Field Planner | 后端服务，根据模板版本规划某次抽取需要落库的字段。 |
| 抽取任务 | ExtractionJob | 一次"把文档内容抽成 EHR 字段"的请求，承载用户意图。 |
| 抽取运行 | ExtractionRun | 抽取任务的一次实际执行实例，承载执行状态、耗时、产物。 |
| 字段当前值 | FieldCurrentValue | 某病例某字段当前最新值（人工审核或抽取产生）。 |
| 字段值证据 | FieldValueEvidence | 字段值的归因证据：原始文本片段 + 文档 ID + 页码/坐标。 |
| 字段值事件 | FieldValueEvent | 字段值的变更历史（谁、何时、从什么改为什么、为什么）。 |
| 科研项目 | ResearchProject | 一个临床研究项目，包含纳入的病例、绑定的 Schema 模板、导出的数据集。 |
| 项目-模板绑定 | ProjectTemplateBinding | 科研项目与 Schema 模板版本的绑定关系。 |
| 项目-病例 | ProjectPatient | 科研项目纳入的病例（多对多关系表）。 |
| 数据上下文 | DataContext | 抽取流程中，传递给 LLM 的上下文打包对象（文档片段、字段定义、历史值等）。 |
| 记录实例 | RecordInstance | Schema 中嵌套分组的实例化（例如"用药记录"有多条）。 |

## 技术术语

| 术语 | 含义 |
|---|---|
| FastAPI | 后端 Web 框架，本项目用 0.109 版本。 |
| SQLAlchemy (async) | ORM，本项目使用异步模式（`asyncpg` / `aiomysql`）。 |
| Alembic | 数据库 migration 工具。 |
| Celery | 异步任务队列，承载 OCR / 抽取 / metadata 等长耗时任务。 |
| LangGraph | LLM 工作流编排库，用于复杂抽取链路。 |
| TextIn | 第三方 OCR 服务，本项目用于 PDF/扫描件识别（含坐标）。 |
| Bases (.base) | Obsidian 1.10+ 自带的数据库视图，能按 frontmatter 自动聚合 md 文档。 |
| Canvas (.canvas) | Obsidian 自带的可视化白板格式（JSON）。 |
