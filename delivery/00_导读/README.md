# EACY 交付文档

> 本文档面向**项目交付方**的技术对接人员，覆盖 **验收 / 二次开发对接 / 运维移交** 三类用途。

## 一分钟读懂 EACY

EACY 是一套面向临床科研的**医疗文档智能处理 + EHR/CRF 数据治理**平台，核心能力：

1. **文档接入** — 支持 PDF/扫描件上传，自动 OCR；
2. **AI 抽取** — 基于 LLM 把非结构化病历文本抽取为结构化字段（EHR），保留**证据归因**（哪一句、哪一页）；
3. **Schema 模板与 CRF** — 用可视化方式定义字段集合，绑定到科研项目；
4. **科研数据集** — 按项目维度组织患者纳入、字段抽取、数据导出；
5. **管理后台** — 任务监控、用户与权限、异步队列运维。

详细业务定位见 [[系统定位与价值]]。

## 文档使用约定

- 本仓库是一个 **Obsidian Vault**，建议用 Obsidian 打开 `/eacy_project/delivery/` 目录获得最佳体验（属性面板、双链、Bases 视图、Canvas）。
- 也可用任何 Markdown 编辑器或 GitHub 网页阅读，但**索引页（.base）和架构图（.canvas）需要 Obsidian 渲染**。
- 每篇文档头部的 YAML 属性 (`frontmatter`) 是文档元数据，含义见 [[文档结构说明]]。
- 文档与代码的同步机制、健康度检查见 [[文档结构说明#文档与代码的同步机制]]。
