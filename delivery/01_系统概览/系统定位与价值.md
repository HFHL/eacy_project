# 系统定位与价值

## 一句话定位

EACY 是面向临床科研的 **"医疗文档 → 结构化 EHR → 科研 CRF 数据集"** 一体化平台，用 OCR 与 LLM 自动化把非结构化病历转为可统计、可导出、可溯源的科研数据。

## 解决的核心问题

传统临床科研数据采集的痛点：

| 痛点 | EACY 的解法 |
|---|---|
| 病例资料散落（纸质、PDF、扫描件），人工录入耗时 | 上传 → OCR → AI 抽取一条龙，**保留页面坐标与原文证据** |
| 不同科研项目字段口径不一，模板硬编码 | **Schema 模板 + 版本化绑定**，前端可视化设计 |
| 抽取结果难审核、来源不可追溯 | 每个字段值挂 [[术语表#字段值证据]]（原文片段 + 文档 ID + 坐标） |
| 修改字段值丢历史 | [[术语表#字段值事件]] 记录完整变更链 |
| 科研项目数据导出难 | 按项目维度组织，一键导出 Excel/CSV |

## 主要使用场景

1. **临床研究中心**：把历史病历批量转为结构化 EHR，再按多个科研项目划分数据集。
2. **多中心研究**：统一 Schema 模板下发到各中心，保证字段口径一致。
3. **回顾性研究**：批量上传纸质病历扫描件，自动抽取，人工校验后入库。

## 关键非功能特性

- **可溯源**：字段值 → 证据片段 → 原始文档 → 页面坐标。
- **可演进**：Schema 模板版本化，绑定到项目的是具体版本号，老数据不破。
- **可观测**：异步任务（OCR、抽取）有独立的进度追踪与管理后台。详见 [[异步任务进度追踪]]（待写）。
- **可扩展**：抽取链路用 LangGraph 编排，新增字段类型 / 新增 LLM 提供商有标准扩展点。

## 不在范围内的事项

为避免误解，明确**本系统不做**：

- 不做病种知识图谱推理（只做"从文本到字段"的抽取）
- 不做临床决策支持（CDSS）
- 不做与院内 HIS / EMR 的实时双向集成（当前以离线文档上传为主）
- 不做患者隐私脱敏（默认数据已合规授权用于科研）
