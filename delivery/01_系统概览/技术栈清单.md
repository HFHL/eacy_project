# 技术栈清单

> 版本以 `backend/pyproject.toml` 和 `frontend_new/package.json` 为准；本文记录**选型理由与对接注意**。

## 后端

| 组件 | 版本 | 角色 | 选型理由 / 对接注意 |
|---|---|---|---|
| Python | 3.11.7 | 运行时 | 异步生态成熟；与 `langgraph` 兼容 |
| FastAPI | ^0.109.1 | Web 框架 | 内置 OpenAPI；与异步 SQLAlchemy 适配良好 |
| Uvicorn / Gunicorn | ^0.25 / ^23 | ASGI 服务器 | 生产用 Gunicorn + Uvicorn worker |
| SQLAlchemy (asyncio) | ^2.0.25 | ORM | 异步模式，与 FastAPI 一致 |
| asyncpg / aiomysql / pymysql | — | DB 驱动 | 同时支持 PostgreSQL 与 MySQL |
| Alembic | ^1.13.1 | DB migration | 见 [[首次部署手册]]（待写）中的初始化步骤 |
| Celery | ^5.3.6 | 异步任务队列 | OCR / 抽取 / metadata 任务均走 Celery |
| Redis | ^5.0.1 | Cache + Celery broker/backend | 部署时单实例即可，详见 [[部署架构]]（待写） |

| httpx | ^0.26 | 外部 HTTP 客户端 | 调用 TextIn OCR / LLM |
| python-multipart | ^0.0.27 | 文件上传 | 文档上传依赖 |

## 数据库

支持 **MySQL** 与 **PostgreSQL** 两种部署形态（驱动同时安装），Alembic migration 共用一套。生产建议 PostgreSQL（与 `asyncpg` 配合性能更好）。详见 [[部署架构]]（待写）。
