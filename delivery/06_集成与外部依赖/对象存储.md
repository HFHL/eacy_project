# 对象存储（阿里云 OSS）

> EACY 用阿里云 OSS 存放**原始上传文档**（PDF / 图片 / Office 等）。OCR / 抽取 / 前端预览全部通过 OSS 签名 URL 读取。**代码层只实现了 OSS**，没有 S3 / 本地磁盘后端。

## 一、用途与触发场景

| 用途 | 触发点 |
|---|---|
| **存原始文件** | `document_service.upload_document` → `DocumentStorage.save` |
| **OCR 输入** | `process_document_ocr` 先拿签名 URL → 交给 [[TextIn-OCR]] 下载 |
| **前端预览** | 文档详情页 / `DocumentDetailModal` → `get_preview_url` |
| **下载** | 同上，附带 `response-content-disposition: attachment;filename=...` |

> 详见 [[端到端数据流]] 阶段 [1]。

---

## 二、接入方式

- **协议**：HTTPS REST，**未使用 OSS Python SDK**，由 `AliyunOssDocumentStorage` 自行：
  - PUT 上传：`http.client.HTTPSConnection`
  - GET 签名 URL：手工拼 `?OSSAccessKeyId=...&Expires=...&Signature=...`
- **签名算法**：HMAC-SHA1 over `METHOD\n\n${content_type}\n${expires_or_date}\n/${bucket}/${key}[?subresources]`，结果 base64
- **工厂方法**：`build_document_storage()` 读 `DOCUMENT_STORAGE_PROVIDER`；目前只接受 `oss`，其它值直接报错（`EACY document storage is OSS-only`）
- **数据契约**：上传后返回 `StoredDocumentFile { provider, path, url, size, sha256 }`，`path` 是 OSS object key，写入 `document.file_path`

### 存储路径策略

```
${OSS_BASE_PREFIX}/YYYY/MM/{uuid4}{file_ext}
```

- 默认前缀 `documents`（可由 `OSS_BASE_PREFIX` 覆盖）
- 按"上传时的 UTC 月份"分目录，避免单层下海量对象
- 文件名是新 UUID，**不复用原文件名**，原文件名只保留在 `document.original_filename` 字段

### 签名 URL 策略

`get_signed_url(key, expires_in=3600, response_content_disposition=None)`：

- 默认有效期 **1 小时**
- 若配置了 `OSS_PUBLIC_BASE_URL`（CDN / 反向代理 / 公共桶），返回**无签名**的直链
- 支持 `response-content-disposition`，用于强制浏览器下载并指定文件名

## 三、关键配置项（环境变量）

| 环境变量 | 含义 | 默认 |
|---|---|---|
| `DOCUMENT_STORAGE_PROVIDER` | 当前只支持 `oss` | `oss` |
| `OSS_ACCESS_KEY_ID` | OSS AK | 无 |
| `OSS_ACCESS_KEY_SECRET` | OSS SK | 无 |
| `OSS_BUCKET_NAME` | Bucket | 无 |
| `OSS_ENDPOINT` | Endpoint，例如 `oss-cn-hangzhou.aliyuncs.com`（缺协议会自动补 `https://`） | 无 |
| `OSS_REGION` | Region 标识（保留，未在代码中读） | 无 |
| `OSS_BASE_PREFIX` | 对象 key 前缀 | `documents` |
| `OSS_PUBLIC_BASE_URL` | 公开访问基址（若设置则签名 URL 退化为直链） | 无 |

> [!warning] 实际值
> AK/SK 不写在文档里；部署方在 `.env` 维护。见 [[环境变量清单]]（待写）。
> 缺任意 OSS 必填项时，`build_document_storage()` 启动时即 fail-fast。

## 四、配额与限速

- OSS 侧 QPS / 单 bucket 带宽：**TBD**（受阿里云账号 / 存储类型限制）
- EACY 侧未做自己的限速；上传是同步 PUT，**会阻塞 FastAPI 事件循环**：单文件较大或网络抖动时会拖慢上传接口的吞吐
- 单文件大小限制：**EACY 代码层无显式上限**（由前端 / Nginx / OSS 共同约束）

## 五、降级 / 重试策略

| 层级 | 行为 |
|---|---|
| **PUT 失败** | `AliyunOssDocumentStorage._put_object` 抛 `RuntimeError: OSS upload failed`，**不重试**；调用方 `document_service.upload_document` 直接报错给前端 |
| **签名 URL 过期** | 上游（TextIn 下载 / 前端预览）失败后由用户重试触发新的签名 URL |
| **OSS 故障** | 无自动降级到本地盘 / 备用 bucket |

## 六、常见错误码与处理

| 现象 | 触发 | 处理 |
|---|---|---|
| `Missing OSS configuration: OSS_ACCESS_KEY_ID, ...` | 启动时配置缺失 | 设置完整 OSS 环境变量后重启 |
| `Unsupported document storage provider` | `DOCUMENT_STORAGE_PROVIDER` 非 `oss` | 改回 `oss`（或等到引入新 backend 时再扩） |
| `OSS upload failed: 403 ...` | AK/SK 错误 / 桶 ACL / 时钟漂移导致签名失效 | 校验 AK、桶权限、服务器时间 |
| `OSS upload failed: 404 NoSuchBucket` | bucket 不存在或 endpoint 与 region 不匹配 | 核对 endpoint / bucket |
| 预览 URL 打开报 `SignatureDoesNotMatch` | 签名计算与服务器时间偏差过大 | 同步 NTP |
| TextIn 下载阶段 `Document download failed before OCR` | 签名 URL 过期 / OSS 网络不通 | 见 [[TextIn-OCR]] 错误处理 |

## 七、离线替代方案

- **代码层没有实现本地磁盘 backend**。若需离线 / 私有化部署：
  - 短期：用 OSS 兼容协议产品（如 MinIO）替代——只要支持 OSS V1 签名头与同名 endpoint 路径，`AliyunOssDocumentStorage` 可直接复用
  - 长期：在 `app/storage/document_storage.py` 新增 `LocalDocumentStorage` 子类（实现 `save` + 提供给 `document_service` 一个"本地预览 URL"），并扩展 `build_document_storage()` 分支（**未实现，TBD**）
