# 业务概述

> [!info] 一句话
> EACY 用户系统是个**最小可用**实现：本地账号 + JWT 双 token + 一个 `role` 字段区分 `admin` / 普通用户。**没有"角色 → 权限点"的完整 RBAC**，写在 `users.permissions` 列里的字符串只是一个手工逗号分隔的能力清单，业务接口几乎不消费它。

## 一、User 模型

数据表 [[表-user]]（字段定义以 `backend/app/models/user.py` 与 Alembic 迁移为准）。核心要点：

- **主键 `id`**：UUID 字符串（`Uuid(as_uuid=False)`，存为字符串）。这一点决定了所有"用户作用域"列（`patient.owner_id`、`document.uploaded_by` 等）也是 UUID 字符串。
- **唯一约束**：`email`、`username` 各自唯一索引。
- **`password_hash`**：`pbkdf2_sha256$<iterations>$<salt>$<digest>` 自定义格式，迭代 120 000 次。**未引入 bcrypt / argon2 库**。
- **`role`**：纯字符串，默认 `"user"`，无 enum 约束。
- **`permissions`**：单列字符串，逗号分隔（`AuthService._permissions_to_list`），存在但**业务路由几乎未消费**——见下一节。
- **`is_active`**：登录前会校验，禁用即拒登。
- **`last_login_at`**：每次硬登录更新（`AuthService.login`）。

## 二、JWT 双 Token 机制

- **签发**：`AuthService.create_token`，HS256，密钥来自 `config.JWT_SECRET_KEY`（默认 `"fastapi"`，**必须在生产环境覆盖**）。
- **payload**：`{user_id, sub, username, name, email, role, permissions, type, iat, exp}`。
- **access_token** 有效期 8 小时（`ACCESS_TOKEN_SECONDS = 60*60*8`）。
- **refresh_token** 有效期 14 天（`REFRESH_TOKEN_SECONDS = 60*60*24*14`），通过 `type=refresh` 区分；`/auth/refresh` 会再签一对新 token。
- **校验入口有两条**，**实际生效的是第二条**：
  1. `backend/core/fastapi/middlewares/authentication.py::AuthBackend` —— Starlette 级 middleware，只把 `payload["user_id"]` 塞到 `request.user.id`，**不抛错**。
  2. `backend/app/core/security.py::decode_authorization_header` + `backend/app/core/auth.py::get_current_user` —— FastAPI 依赖，**业务路由实际依赖这条**，会抛 401。

## 三、当前权限策略（实测）

> [!warning] 不要把 `users.permissions` 当成 RBAC
> 代码里只有 `require_permissions(...)`（`app/core/auth.py`）会检查 `current_user.permissions` 是否覆盖所需点位，但**搜遍业务路由没有任何一处调用 `require_permissions`**。换句话说目前是"登录即可用"。

实测有效的访问控制只有三条：

| 控制点 | 实现 | 行为 |
|---|---|---|
| **登录态** | `Depends(get_current_user)` | 凡是依赖它的接口，无 token 且 `ENABLE_AUTH=true` 时返回 401 |
| **管理员守门** | `app/core/auth.py::is_admin_user` → `/admin/*` 路由的 `require_admin_user` | `role == "admin"` 或 `permissions` 含 `"*"` 才放行，否则 403 |
| **数据作用域** | 仓储层 `owner_id` / `uploaded_by` 过滤（详见 [[关键设计-鉴权与作用域]]） | 不同用户查不到对方上传的文档 / 创建的病例 |

`backend/core/fastapi/dependencies/permission.py` 里另有一组 `IsAuthenticated / IsAdmin / AllowAll / PermissionDependency`，是 starter template 自带类，**当前业务路由未使用**，可视为"备用骨架"。

## 四、接口面（业务说明）

> 接口字段、请求体、状态码请直接看 OpenAPI / `backend/app/api/v1/auth/router.py`。

| 接口 | 业务含义 | 关键边界 |
|---|---|---|
| `POST /auth/register` | 新建本地账号，**返回值就是 TokenResponse**（注册即登录） | 邮箱、用户名唯一约束冲突 → 409 |
| `POST /auth/login` | 邮箱 + 密码登录 | 用户不存在 / 已禁用 / 密码错 → 统一 401，**不区分原因** |
| `POST /auth/refresh` | 用 refresh_token 换新一对 token | refresh_token 类型不是 `refresh` 或用户被禁用 → 401 |
| `POST /auth/logout` | **当前实现是 no-op**（无服务端会话/黑名单），客户端自行清掉本地 token | 仅做"在线已登录"前置校验 |
| `GET /auth/me` | 返回 `get_current_user` 注入的 `CurrentUser` | 开发期 `ENABLE_AUTH=false` 会返回 `dev_admin` |

> [!info] 注册验证码 / 微信扫码 / 重置密码
> 前端 `Login.jsx` 有完整 UI 与表单字段，但 `frontend_new/src/api/auth.js` 中 `sendRegisterEmailCode / sendResetPasswordEmailCode / resetPasswordByEmail / getWechatQrCode` 全是 `emptySuccess(...)` 桩，**后端没有实现对应接口**。`code` 字段在注册表单里被前端硬校验为 `0000`。这些都标 **TBD**。
