# 验收要点

> 5 条最小验收用例，覆盖：访问控制、列表筛选、详情下钻、停滞探测、与业务页同步。详细任务进度阶段验收见内部规划文档 `eacy/EACY开发计划 7.1 管理后台任务监控与实时进度验收方案.md`（不在交付库内）。

## A1 · 访问控制

> [!example] 操作
> 用非管理员账号请求 `GET /api/v1/admin/extraction-tasks`

- [ ] 返回 `403 Admin permission required`
- [ ] 管理员账号同请求返回 200，结构包含 `items / total / type_counts / status_counts`

## A2 · 列表筛选与计数

> [!example] 操作
> 进入 `/admin` 抽取任务 Tab，切换"科研 CRF / 电子病历夹 / 靶向"分段筛选，并叠加状态下拉

- [ ] `type_counts` 与 `status_counts` 数值与列表实际行数一致（含 `all`）
- [ ] 关键字搜索覆盖 ID / 患者名 / 项目名 / 模板名 / 目标表单
- [ ] 切换分段不会丢失搜索关键字（前端 useMemo 已实现）

## A3 · 详情下钻

> [!example] 操作
> 选中一条已完成的 batch，点击"详情"

- [ ] `summary` 展示项目 / 患者 / 模板 / 进度 / 计数 / 错误摘要
- [ ] `jobs[]` 中每行可展开 `extracted_fields`，其中至少一行带有 `source_text + source_page`（来自 [[AI抽取/证据归因机制]]）
- [ ] `llm_calls[]` 数量等于 `validation_log` 非空的 run 数；右上角显示 `llm_source = run`
- [ ] 不存在的 `task_id` 返回 404 `Admin extraction task not found`

## A4 · 停滞探测

> [!example] 操作
> 人为构造一个 `running` 状态但 `heartbeat_at` / `updated_at` 早于 15 分钟前的 batch（例如直接 UPDATE 数据库）

- [ ] Admin 列表展示该 batch 状态为 `stale`（橙色 / 警告色）
- [ ] DB 中 `async_task_batches.status` **仍是** `running`（确认归一化只发生在读侧）
- [ ] 状态下拉中 `已停滞` 计数 ≥ 1

## A5 · 与业务页同步

> [!example] 操作
> 在患者详情或科研 CRF 页面触发一次"更新电子病历夹/CRF"，同时打开 Admin

- [ ] 业务页轮询 `/task-batches/{batch_id}` 与 Admin 列表行 `progress / succeeded_items / failed_items` 在 ~5 秒内一致
- [ ] running 状态下 Admin 详情建立 SSE 连接（顶部 Badge 显示"实时接收中"），收到 terminal 事件后自动刷新详情
- [ ] 任务全部完成后业务页 EHR/CRF 数据可见新值；Admin 顶部 `active_tasks` 数下降
